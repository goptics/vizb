# AGENTS.md

Vizb is a Go CLI that turns benchmark output (Go / Rust / JavaScript) and
tabular CSV/JSON data into a single self-contained HTML visualization (Vue 3 +
echarts + echarts-gl, embedded in the Go binary). See `README.md` for the
public surface and `action.yml` for the GitHub composite action.

## Build & test

Task runner is [`task`](https://taskfile.dev/).

```bash
task init          # one-time: go mod tidy + pnpm install (ui + docs) + act + lefthook
task build         # ui + cli + docs
task build:ui      # SINGLEFILE=True pnpm build inside ui/ — regenerates pkg/template/vizb-ui.gen.go
task build:cli     # go build to ./bin/vizb
task test          # go test -count=1 ./... + pnpm test in ui/
task lint          # golangci-lint run
task format        # gofmt -w . + pnpm format in ui/
task dev:ui        # vite dev server
task dev:docs      # astro dev server on :4321
```

Single Go test: `go test -run TestName -v ./path/to/package` (no Taskfile shortcut).

## Architecture (one line per area)

- `main.go` → `cmd/` cobra commands (`root`, `ui`/`html`, `merge`, plus chart subcommands)
- `cmd/charts/{bar,line,pie,heatmap,radar}/` self-register via `cli.Register(NewCommand)` in `init()`; `cmd/root.go` blank-imports each one
- `cmd/cli/` shared pipeline (`pipeline.go`), composable options (`options.go`), registry (`command.go`)
- `pkg/parser/{csv,golang,javascript,json,rust}/` self-register parsers in a `map[string]ParseFunc`; `pkg/parser/detect.go` auto-sniffs format from content (with `go` as final fallback)
- `pkg/template/` HTML generation + chunk pruning; embeds the built Vue UI from `vizb-ui.gen.go`
- `pkg/style/` lipgloss terminal styling
- `shared/` cross-package types (`Dataset`, `DataPoint`, `DatasetSettings`), temp-file manager, merge logic
- `ui/` Vue 3 + Vite + vitest; built with `SINGLEFILE=True` to produce a single inlined HTML
- `docs/` Astro site (separate `pnpm` workspace, built independently)
- `action.yml` GitHub composite action — downloads the binary from GitHub Releases

## Things agents get wrong

- **`pkg/template/vizb-ui.gen.go` is generated.** Header: "Code generated by vite.config.ts; DO NOT EDIT." Regenerate with `task build:ui` after any change under `ui/src/`. The lefthook pre-commit hook (`ui-gen-check` in `.lefthook.yml`) blocks any commit that touches `ui/src/**` without also staging the regen — run `task build:ui` before `git commit`.
- **HTML template delimiters are `[[VIZB ... VIZB]]`, not `{{ }}`.** echarts-gl's clay.gl GLSL shaders embed literal `{{ }}` for loop unrolling, which would collide with `html/template`. The Go side parses with `.Delims("[[VIZB", "VIZB]]")` (`pkg/template/generate-ui.go`); the matching placeholder is in `ui/vite.config.ts` (`appendVizbDataScriptTag`). If you see the literal text `[[VIZB .Data VIZB]]` in the output, a delimiter change broke the wiring.
- **Chart subcommands and parsers self-register.** Adding a chart type = new `cmd/charts/<name>/` package with `func init() { cli.Register(NewCommand) }` + blank import in `cmd/root.go`. Adding a parser = new `pkg/parser/<name>/` package with `Parsers["key"] = fn` in `init()` + blank import in `cmd/root.go`. Don't add charts/parsers in a different way.
- **`--scale` is per-chart (bar, line) only** — never a root flag. Pie/heatmap/radar don't expose it (enforced at compile time via separate `Options` structs in `cmd/charts/{bar,line}/`).
- **Output format is inferred from the `-o` extension** (`.json` → JSON, anything else → HTML). No flag toggles it; `cmd/cli/output.go::InferFormatFromExtension` is the single source of truth.
- **Grouping**: `-p/--group-pattern` (slash/underscore delimited) or `-r/--group-regex` (named captures). For CSV/JSON, `-g/--group` lists the column/field names backing each dimension; for benchmark parsers it lists human-readable axis labels.
- **Custom options pattern** — new flags belong in `cmd/cli/options.go` (`CommonOptions` → `LinearOptions` → `ChartOptions`) with a `Bind(*pflag.FlagSet)` method. Don't add new mutable globals to `shared/FlagState`.
- **Temp files** go through `shared.TempFiles` (`sync.Map`), cleaned up via `defer shared.TempFiles.RemoveAll()` in `cmd.Execute`. Use `shared.MustCreateTempFile` / `shared.MustCreateFile`, not `os.CreateTemp` / `os.Create`.

## Testing

- Go: `go test -count=1 ./...` (Taskfile also runs `pnpm test` in `ui/`).
- UI vitest: pure-function unit tests only. `ui/vitest.config.ts` uses `environment: 'node'` and explicitly excludes the single-file build plugins from `vite.config.ts`. Don't add browser/DOM tests there.
- **CLI tests use `testify/suite` only** — `s.Equal` / `s.Require.*` / `s.Panics`; no bare `testify/assert` or `testify/require` on `*testing.T`. Subtests use `s.Run`, not `t.Run` + `assert`.
- **Test method names are PascalCase** after `Test` (no underscores): `TestBarCommandNewShape`, not `TestBarCommand_NewShape`.
- **Shared helpers** live in [`testutil/`](testutil/helpers.go): `TrapOsExitPanic`, `WriteBenchFile`, `WriteJSON`, `ReadDataset`, `CaptureStdout`/`CaptureStderr`. Flag resets use `cmd.ResetTestState()` from [`cmd/testing.go`](cmd/testing.go). `TrapOsExitPanic` is implemented in [`shared/trap_os_exit.go`](shared/trap_os_exit.go) (avoids import cycle); `shared/` tests call it directly, `cmd/` tests use `testutil.TrapOsExitPanic`.
- **Fatal paths** trap `shared.OsExit` via `TrapOsExitPanic`, then `s.Panics` + assert `*exitCalled`.
- **Shared unit tests** (`shared/*_test.go`, `shared/utils/*_test.go`): `testify/suite` only; PascalCase `Test*` methods; table cases via `s.Run`. External tests in `shared_test` package (`dataset_test.go`, `migrate_external_test.go`) follow the same suite pattern.
- **Chart e2e** (`cmd/charts/*/*_test.go`): `NewCommand()` → `-o out.json` → `testutil.ReadDataset` → assert `Settings[i].ChartType()` and typed config. Mirror [`cmd/charts/bar/bar_test.go`](cmd/charts/bar/bar_test.go) (four roles: `TestCommandFlags`, `TestBakes*OnlySelection`, `Test*CommandNewShape`, `Test*CommandBadSwapExits`).
- **Chart config unit** (`config/charts/*/*_test.go`): `testify/suite` only; `TestMaterialise*Precedence` + chart-specific invariant test (e.g. `TestPieConfigNoScaleOrAutoRotate`). Registry/decode tests live in [`config/charts/bar/bar_test.go`](config/charts/bar/bar_test.go).
- **Subcommand e2e** (`cmd/ui_test.go`, `cmd/merge_test.go`, `cmd/root_test.go`): `rootCmd.SetArgs` + `Execute()`; call `ResetTestState()` in `SetupTest`.

## Conventions

- Format: `gofmt` + `prettier --write .` in `ui/`. Run `task format` before committing.
- Do not co-author AI in commit messages; keep the project author credit in PRs.
- Releases: GoReleaser builds with `CGO_ENABLED=0` for linux/darwin/windows × amd64/arm64. `version.Version` is ldflag-injected (`-X github.com/goptics/vizb/version.Version={{.Tag}}`). Snapshot release via `task release:snapshot`; tagged release via `task release VERSION=vX.Y.Z` (signs the tag with `-s`).
- Test the GitHub Action locally with `task act:test:stateless` or `task act:test:stateful` (requires `act` from `task init`).
