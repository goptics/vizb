# Project

**vizb** â€” Go CLI tool that transforms `go test -bench` output into interactive 4D HTML visualizations (Vue 3 + ECharts). Outputs a single deployable HTML file with embedded data.

Go 1.24+, module: `github.com/goptics/vizb`.

## Architecture

```
main.go â†’ cmd.Execute()
  cmd/               Cobra CLI (root, merge, html subcommands)
  shared/            Domain types, merge logic, OS helpers, flag state
  pkg/parser/        Benchmark name parsing (pattern & regex grouping)
  pkg/style/         CLI terminal styling (lipgloss)
  pkg/template/      HTML generation â€” embeds built Vue UI
    ui/              Vue 3 + Vite + Tailwind + ECharts SPA
  version/           Build-info version string
```

Key data types live in `shared/bench.go`: `Benchmark`, `BenchmarkData`, `Stat`, `HistoryEntry`.

Global flag state is held in `shared.FlagState` (single mutable struct â€” all commands read/write it).

## Testing

- Table-driven tests via `testify`. Run single: `go test -count=1 -v ./cmd -run TestName`
- Tests use `os_test.go` / `safe_test.go` to stub `os.Exit` and filesystem calls â€” do not call `os.Exit` directly in code; use `shared.OsExit` so tests can intercept.
- CI runs `go test -coverprofile=coverage.txt -covermode=atomic ./...` with Codecov upload.

## UI Development

The Vue SPA lives in `pkg/template/ui/`. It is built into a single HTML file via `vite-plugin-singlefile`, then embedded into the Go binary through `pkg/template/vizb-ui.gen.go`.

- **Never edit** `vizb-ui.gen.go` directly â€” it is generated by the Vite build.
- UI build requires `SINGLEFILE=True` env var (set by `task build:ui`).
- UI deps: Vue 3, ECharts 5, radix-vue, Tailwind 3, TypeScript.
- Dev server: `task dev:ui` (opens browser via Vite).

## CLI Subcommands

- **`vizb [target]`** (root) â€” parse benchmark file or stdin, output HTML/JSON
- **`vizb merge [files...]`** â€” merge multiple benchmark JSON files with tag-based smart merging
- **`vizb html [file]`** â€” generate HTML from an existing benchmark JSON

## GitHub Action

`action.yml` is a composite action. Local testing uses `act` via `task act:test:stateless` / `task act:test:stateful`. Workflow configs in `.github/workflows/`: `ci.yml` (tests), `release.yml` (goreleaser), `test-action.yml` (act), `deploy-examples.yml`.

## Conventions

- Errors: use `shared.ExitWithError(msg, err)` â€” prints styled message and exits.
- Temp files: use `shared.MustCreateTempFile(prefix, ext)` and track via `shared.TempFiles.Store()` for cleanup.
- File creation: use `shared.MustCreateFile(path)` / `shared.MustOpenFile(path)` wrappers (handle errors consistently).
- Release: GoReleaser builds for linux/darwin/windows on amd64+arm64. Triggered by git tags.
