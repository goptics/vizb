When a **z-axis** is present and the active chart is **bar** or **line**, vizb automatically switches to a visual WebGL 3D scene instead of a flat 2D chart. Rotate, zoom, and pan directly in the browser. Other chart types (pie, radar, heatmap) receive z-axis data differently — they don't render a 3D scene.

## What Makes a Chart 3D

A chart renders in 3D when a data point has **all three** plotting axes present:

- **XAxis** — categories along the width
- **YAxis** — categories along the depth's base
- **ZAxis** — the depth/height dimension stacked on top

The **Name** dimension stays optional — it still groups points into separate
charts. Without a z-axis, charts render as the usual 2D bar/line/pie.

<Aside type="note">
  The z-axis (`z`) is only valid alongside **both** `x` and `y`. A 3D chart needs
  an x/y floor to stand on.
</Aside>

## Generating 3D Data

Add a `z` segment to your grouping pattern or a `(?<z>…)` capture to your regex.

**Pattern — `x/y/z`:**

```bash
vizb data.csv -g category,metric,group --group-pattern "x/y/z" -o output.html
```

| Source Label | Name | XAxis | YAxis | ZAxis |
|---|---|---|---|---|
| `BenchmarkSort/1024/QuickSort/warm` | Sort | 1024 | QuickSort | warm |
| `BenchmarkSort/2048/MergeSort/cold` | Sort | 2048 | MergeSort | cold |

The same `z` dimension can come from a CSV or JSON column instead of a benchmark
name segment — pass the column as the fourth `--group` field. See the
[Tabular Data guide](/guides/data).

**Regex — named `z` capture:**

```bash
vizb data.csv -g category,metric,group,series --group-regex "(?<n>.*)/(?<x>.*)/(?<y>.*)/(?<z>.*)" -o output.html
```

If you supply a `z` without both `x` and `y`, vizb rejects the pattern:

```text
zAxis (z) requires both xAxis (x) and yAxis (y)
```

See the [Grouping Guide](/guides/grouping) for full pattern and regex syntax.

## Axis-Sum Tooltips

Hover any cell to see its breakdown plus **marginal sums** — each marginal is the
sum over the other axes:

- **Σ z** — the stacked height at this `(x, y)` cell (sum over z)
- **Σ x** — total for this x value (sum over the other axes)
- **Σ y** — total for this y value (sum over the other axes)

Sums honor the legend: toggle a z series off and the totals recompute to reflect
only what's visible. Tooltips follow light/dark mode automatically.
