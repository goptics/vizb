The **bar chart** is vizb's default chart type. It maps categories to bars, making it easy to compare values at a glance — whether you have one series or a full three-dimensional dataset. As you add dimensions, bars group and then grow into a full 3D WebGL scene.

Use bar charts when you want direct side-by-side comparison of discrete categories, benchmark names, or labeled data points. If you're already using bar and want to fold in a z-axis without the depth, the [Heatmap](/charts/heatmap) is a good complement.

## How vizb builds it

vizb maps your grouping columns to the chart's visual axes:

| Dimension | Role |
|-----------|------|
| **XAxis** | Category axis — each unique X value becomes a bar group |
| **YAxis** | Series axis — each unique Y value becomes a separate bar within each X group |
| **ZAxis** | Depth layer — stacks Z values into the X/Y floor grid (3D only, requires echarts-gl) |

## Dimensions

### 1D (X axis)

A single series of bars — one bar per X category. The height is the raw value. This is the simplest view: run a benchmark, pick one grouping column, get a bar per result.

```bash
vizb bar data.csv -g category -o out.html
```

### 2D (X + Y axes)

Grouped bars: X stays on the category axis, and each distinct Y value becomes its own bar series side-by-side within each X group. The legend lists the Y values so you can toggle individual series on and off.

```bash
vizb bar data.csv -g category,metric -p x,y -o out.html
```

### 3D (X + Y + Z axes)

WebGL `bar3D` scene: X and Y form a grid floor, and Z values stack as depth layers on each (X, Y) cell. Rotate, zoom, and pan the scene in the browser. Requires echarts-gl (bundled automatically).

```bash
vizb bar data.csv -g category,metric,group -p x,y,z -o out.html
```

## `--select` (csv/json)

`--select` is **repeatable** and plays three roles depending on whether grouping is explicit. See the [mode matrix](/commands/root#--select-mode-matrix) for a full comparison.

### Role 1 — Grouped + numeric select

With explicit `-g` / `-p` / `-r`, `--select` picks which **numeric columns** become bar series. Optional `{label}` renames series.

```bash
vizb bar sales.csv -g region,product -p x,y --select amount,total -o out.html
```

### Role 2 — Solo all-numeric value axes

Without grouping, each `--select` defines one **value-mode** view: 2–3 numeric columns become continuous `x`, `y`, and optional `z` axes. Auto-group and auto-value are disabled. With 3 columns, bar auto-enables 3D (`bar3D`).

```bash
# Two numeric columns → 2D value bars
vizb bar spiral-3d.csv --select x,y -o bar.html

# Three numeric columns → 3D value bars
vizb bar spiral-3d.csv --select x,y,z -o bar3d.html
```

### Role 3 — Solo mixed category + value

Without grouping, when the first selected column is categorical and the rest are numeric, vizb parses **mixed-axis** data (category x + value y[,z]). Bar accepts this shape at parse time; [scatter](/charts/scatter) is the primary chart for category × value plots.

```bash
vizb bar examples/csv/region-metrics.csv --select region,latency -o bars.html
vizb scatter examples/csv/region-metrics.csv --select region,latency -o scatter.html
```

Repeat `--select` for multiple datasets in one HTML file (dataset tabs in the UI).

## Auto-value mode (all-numeric data)

When every column in a CSV/JSON file is numeric (no categorical columns), vizb auto-detects the first 2–3 columns as coordinate axes and renders them as value-type bars — no `--group` needed. With 3+ columns, bar auto-enables 3D (`bar3D`).

```bash
# Two columns → x, y bars
vizb bar prices.csv -o bar.html

# Three columns → x, y, z bar3D
vizb bar spiral.csv -o bar.html
```

Solo `--select` overrides auto-value: it disables inference and uses only the columns you name. With 2 columns selected, the chart stays 2D even if the file has more numeric columns.

## Settings

These settings apply to bar charts:

| Setting | CLI flag | UI toggle | Notes |
|---------|----------|-----------|-------|
| Sort | `--sort asc\|desc` | Sort control | Sorts bars along the X axis |
| Labels | `--show-labels` | Show labels | Displays value on each bar |
| Swap | `--swap` | Axis switcher | Rotates which column maps to X vs Y |
| Scale (log) | `--log-scale` | Scale toggle | Log Y axis — 2D only |
| 3D view | `--3d` | 3D view | Value 3D for x+y data (y → depth, metric → height) |
| 3D visual map | `--3d-visualmap` | 3D visual map | Metric gradient coloring — on by default with `--3d`; grouped/value 3D |
| Auto-rotate | `--3d-rotate` | Auto rotate | Spins the 3D scene — 3D only |

Override settings for just this chart type without affecting others:

```bash
# Sort ascending and show labels on bars only
vizb bar data.csv -g category,metric --sort asc -l -o out.html

# Log scale on the bar chart only
vizb bar data.csv -g category,metric --scale log -o out.html
```
