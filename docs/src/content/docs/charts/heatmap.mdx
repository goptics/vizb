The **heatmap** packs X and Y into a flat grid of colored cells. Each cell shows a value and a color — spot high/low combinations at a glance across the full matrix.

Use heatmaps when you have two categorical dimensions. A z-axis is optional: without z you get a single-gradient matrix; with z each cell sums its z-series values and blends their colors.

## How vizb builds it

| Dimension | Role |
|-----------|------|
| **XAxis** | Columns of the grid |
| **YAxis** | Rows of the grid |
| **ZAxis** | Folded into each cell: cell **value** = `Σ z`, cell **color** = value-weighted blend of z-series colors |

A cell with one z-series shows that series' pure color. A cell mixing two z-series blends them proportionally — color reads as composition while the number reads as magnitude.

## Dimensions

### 2D (X + Y axes)

Classic heatmap: one value per X × Y cell, colored on a continuous gradient with a `visualMap` scale. No z-series, no blending — magnitude as color.

```bash
vizb heatmap data.csv -g category,metric -p x,y -o out.html
```

### 3D (X + Y + Z axes)

Z values are folded into each cell rather than rendered as depth. Each cell sums its z-series values and blends their palette colors by contribution weight. Toggle z-series in the legend — cells recompute and re-blend live.

```bash
vizb heatmap data.csv -g category,metric,group -p x,y,z -o out.html
```

  The y-axis stays flat intentionally. Splitting each row by z made the grid tall and sparse — so z is folded into the cell instead.
</Aside>

## Settings

These settings apply to heatmap charts:

| Setting | CLI flag | UI toggle | Notes |
|---------|----------|-----------|-------|
| Sort | `--sort asc\|desc` | Sort control | Orders the matrix axes |
| Labels | `--show-labels` | Show labels | Shows the summed value inside each cell |
| Swap | `--swap yx` | Axis switcher | Swaps X and Y axes |

Override settings for just the heatmap without affecting other charts:

```bash
# Show per-cell labels
vizb heatmap data.csv -g category,metric,group -p x,y,z -l -o out.html

# Sort the axes descending
vizb heatmap data.csv -g category,metric,group -p x,y,z --sort desc -o out.html
```

<Aside type="note">
  `scale` (log) and `3d-rotate` are not available for heatmap — those options are bar and line only.
</Aside>
