Vizb does two distinct jobs: chart **any tabular CSV/JSON data**, and visualize **benchmark
output** from Go, Rust, and JavaScript. Either way, the input format is **auto-detected from
content** — `--parser`/`-P` is optional and only needed to override the guess.

<Aside type="tip">
  Just run `vizb <input> -o output.html`. vizb inspects the content, picks the parser, and prints
  its choice (`✨ Auto-detected parser: …`). See [Auto-Detection](/guides/auto-detection).
</Aside>

## Visualize Tabular Data

Turn any CSV or JSON table into charts — numeric columns/fields become series.

<Tabs syncKey="data">
  <TabItem label="CSV" icon="seti:csv">

  ```bash
  vizb data.csv -o output.html
  ```

  Use `--group`/`-g` to turn non-numeric columns into the name / X / Y dimensions:
  ```bash
  vizb data.csv -g name -o output.html
  ```

  </TabItem>
  <TabItem label="JSON" icon="seti:json">

  A JSON array of objects (each object a row, each numeric key a series). Nested objects flatten to dotted keys.

  ```bash
  vizb data.json -o output.html
  ```

  Group by one or more fields with `--group`/`-g`:
  ```bash
  vizb data.json -g name -o output.html
  ```

  </TabItem>
</Tabs>

<Aside type="note">
  A `go test -bench -json` event stream is **not** treated as tabular JSON — it stays a Go
  benchmark. Full rules in the [Tabular Data guide](/guides/data).
</Aside>

## Visualize Benchmarks

Pipe benchmark output straight in. The framework is detected from the content, so no `--parser` is
needed in most cases.

<Tabs syncKey="lang">
  <TabItem label="Go" icon="seti:go">

  **Pipe**
  ```bash
  go test -bench . | vizb -o output.html
  ```

  **From a saved file**
  ```bash
  go test -bench . > bench.txt
  vizb bench.txt -o output.html
  ```

  **JSON event stream**
  ```bash
  go test -bench . -json | vizb -o output.html
  ```

  Save as JSON for later merging:
  ```bash
  vizb bench.txt -o output.json
  vizb output.json -o output.html
  ```

  <Aside type="note">
    To build from source, **Go 1.24+** is required. Pre-built binaries from the [installation page](/installation) need no Go runtime.
  </Aside>

  </TabItem>
  <TabItem label="Rust" icon="seti:rust">

  Criterion and Divan output are both auto-detected from `cargo bench`:

  ```bash
  cargo bench | vizb -o output.html
  ```

  Force a specific parser with `-P` if needed (`rs:criterion` or `rs:divan`):
  ```bash
  cargo bench | vizb -P rs:divan -o output.html
  ```

  </TabItem>
</Tabs>

<Aside type="tip">
  JavaScript frameworks (Vitest, Tinybench) and every supported format are covered in the
  [Parser Guide](/guides/parsers) — including the metrics each one extracts.
</Aside>
