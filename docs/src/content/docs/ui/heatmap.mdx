The **heatmap** is vizb's fourth chart type. It packs the X, Y, and Z dimensions
into a single flat grid: cells sit on an **X × Y** matrix, each cell's **value sums
the z-axis**, and each cell's **color blends the z-series palette**. It's the compact
alternative to a [3D chart](/ui/3d-charts) when a rotating scene is more than you need.

## How the Dimensions Map

| Dimension | Role in the heatmap |
|-----------|---------------------|
| **XAxis** | Columns of the grid |
| **YAxis** | Rows of the grid (plain — *not* subdivided by z) |
| **ZAxis** | Folded into each cell: the cell **value** is `Σ z`, the cell **color** is a value-weighted blend of the z-series colors |
| **Name** | Still groups points into separate charts, same as the other chart types |

A cell containing only one z-series shows that series' pure palette color. A cell
mixing two z-series shows their blend, leaning toward whichever contributes more
value — so color reads as composition while the number reads as magnitude.

<Aside type="note">
  The y-axis stays flat on purpose. Splitting each row by z made the grid tall and
  sparse, which defeats the point of a heatmap — so z is folded into the cell color
  and value instead.
</Aside>

## Toggling Z-Series

Each z value appears as a **legend** entry at the top of the chart. Toggle one off
and the grid recomputes live — cells re-sum and re-blend using only the visible
z-series, mirroring how the 3D charts respond to legend toggles.

## Without a Z-Axis

When the data has no z dimension, the heatmap falls back to a **classic single-series
heatmap**: one value per X × Y cell, colored on a continuous gradient with a
`visualMap` scale beside the grid. No legend, no blending — just magnitude as color.

## Enabling the Heatmap

The heatmap is on by default. Control it like any other chart type:

```bash
# Only the heatmap
vizb bench.txt -c heatmap -o output.html

# Heatmap alongside others
vizb data.csv -g region,product,month -c bar,heatmap -o output.html
```

Or toggle its visibility from the UI's chart settings. See [Settings](/ui/settings).

<Aside type="tip">
  Per-cell value labels follow the global **show-labels** toggle (off by default).
  Turn them on with `--show-labels` or in the UI when the grid is small enough to read.
</Aside>
