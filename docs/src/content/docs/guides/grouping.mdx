## What is Grouping

Grouping turns each row of your data into a point with up to four dimensions — **Name**, **XAxis**, **YAxis**, and **ZAxis** — so vizb can chart it meaningfully instead of as a flat list.

It always works in **two stages**:

1. **Build a label** for the row. For **CSV/JSON**, you pick the columns with `--group` and vizb joins their per-row values using the **same separators** declared in `--group-pattern`. For **benchmarks**, the benchmark name *is* the label already (e.g. `BenchmarkSort/1024/QuickSort`).
2. **Split the label** into dimensions with `--group-pattern` (separator-based) or `--group-regex` (named captures). This stage is identical for both inputs.

The two flows differ only in stage 1 — where the label comes from — so the same pattern and regex syntax serves both.

## Why Grouping

Grouping creates multi-dimensional charts. Without it, every row is an isolated bar or point. With it, vizb can:

- **Separate charts** by logical category (e.g. one chart per algorithm or region)
- **Populate axes** with meaningful values (e.g. input size on the X-axis)
- **Create series** for variants (e.g. different data distributions on the Y-axis)

| Without Grouping | With Grouping |
|---|---|
| Single flat chart with all rows | Multiple charts grouped by name |
| No axis structure | X-axis and Y-axis with meaningful categories |
| Hard to compare across dimensions | Easy side-by-side comparison |

## The Dimensions

Grouping extracts up to four dimensions from each label:

| Dimension | Role | Description |
|---|---|---|
| **Name** (`name` / `n`) | Groups rows into separate charts | Rows sharing the same name appear in the same chart |
| **XAxis** (`xAxis` / `x`) | Sets the X-axis categories | Each unique value becomes a tick on the X-axis |
| **YAxis** (`yAxis` / `y`) | Creates series / variants | Each unique value becomes a separate series or bar group |
| **ZAxis** (`zAxis` / `z`) | Adds a depth axis → 3D chart | Each unique value becomes a stacked depth layer in a [3D chart](/charts/3d) |

<Aside type="note">
  At least one axis (`x` or `y`) is required for grouping to take effect. The `name` dimension is optional — when omitted, all rows go into a single chart. The `z` dimension requires **both** `x` and `y` to be present.
</Aside>

## Grouping CSV/JSON data

CSV and JSON rows have no benchmark name, so you tell vizb which columns carry the structure. The **`--group` / `-g`** flag selects columns; **`--group-pattern` / `-p`** assigns each column to a dimension. By default each column maps to one dimension (`-p y,x`). Use **bracket slots** `[...]` when a single column's **cell value** encodes multiple dimensions (dates, slash paths, underscore IDs, etc.).

### Matching separators

For tabular data, **`-g` and `-p` must agree on separators**:

| `--group` style | Matching `-p` | Row label (example) |
|---|---|---|
| Comma list (`-g region,product`) | Commas (`-p y,x`) | `USA,Widget` |
| Quoted string (`-g "region product"`) | Spaces (`-p "y x"`) | `USA Widget` |
| Structured (`-g region,product/category`) | Same shape (`-p y,x/z`) | `USA,Widget/EU` |

Comma-separated `-g` (the usual pflag form) **requires** comma-separated `-p`. Passing `-g region,product,month -p x/y/z` is rejected — use `-p x,y,z` instead.

Take this `sales.csv`:

| region | product | sales | latency |
|---|---|---|---|
| USA | Widget | 100 | 12 |
| USA | Gadget | 80 | 15 |
| EU | Widget | 60 | 11 |

```bash
vizb sales.csv --group region,product -p y,x -o sales.html
```

For the first row, vizb joins the selected columns into the label `USA,Widget`, then the pattern `y,x` splits it:

| Selected columns | Label | Pattern `y,x` | Result |
|---|---|---|---|
| `region`, `product` | `USA,Widget` | `y` → `USA`, `x` → `Widget` | YAxis = USA, XAxis = Widget |

**Space-separated columns** — pass one quoted `-g` value and use spaces in `-p`:

```bash
vizb sales.csv -g "region product" -p "y x" -o sales.html
```

**Structured grouping** — embed `/` or `_` inside `-g` to declare a hierarchy; `-p` must mirror it:

```bash
vizb sales.csv -g region,product/category -p y,x/z -o sales.html
```

### Bracket slots — split a column's cell value

When one column holds an encoded string (not separate columns), wrap the inner pattern in **`[...]`**. Each **top-level** slot lines up 1:1 with a `-g` column:

| Slot form | Meaning |
|---|---|
| `x`, `y`, `n`, `z` | Whole cell value → that dimension |
| `x{label}` | Whole cell → dimension, axis titled **label** (overrides column name) |
| `[inner]` | Split the cell value with **inner** separators → multiple dimensions |
| `n{year}-y{months}` inside `[...]` | Split value **and** set per-dimension axis titles |

Top-level separators between slots still match `-g` (commas for `-g a,b`, spaces for `-g "a b"`). **Benchmarks do not use brackets** — keep flat `-p n/x/y` for `bench.txt`.

<Aside type="caution">
  **Quote `-p` when it contains brackets or braces.** In zsh, unquoted `[...]` is a filename glob and `{...}` triggers brace expansion. Use single quotes:

  ```bash
  ./bin/vizb heatmap sales.csv -g order_date,category -p '[n{year}-y{months}-x{dates}],z{category}' -o out.html
  ```
</Aside>

**Custom axis titles with `{label}`** — append `{title}` after any dimension token to override the default axis label (column name for CSV/JSON, or `--group` entry for benchmarks):

```bash
vizb sales.csv -g order_date,category -p '[n{year}-y{months}-x{dates}],z{category}' -o sales.html
```

| Column | Cell | Slot | Axis titles |
|---|---|---|---|
| `order_date` | `2022-2-30` | `[n{year}-y{months}-x{dates}]` | name→**year**, y→**months**, x→**dates** |
| `category` | `Widget` | `z{category}` | z→**category** (not the raw column name `category` unless you want that) |

Works on **plain slots** and **flat patterns** too — no brackets required:

```bash
vizb sales.csv -g region,product -p 'y{Region},x{Product}' -o sales.html
vizb bench.txt -p 'n{Algorithm}/x{Size}/y{Variant}' -o out.html
```

`{label}` overrides the column/`--group` label for that dimension; omit `{}` to keep the column name as the axis title.

**Date + category** — split `2022-2-30` into year/month/day, keep category whole:

```bash
vizb sales.csv -g date,category -p "[x-y-n],z" -o sales.html
```

| date | category | Pattern | Result |
|---|---|---|---|
| `2022-2-30` | `Widget` | `[x-y-n]` splits on `-`, `z` ← category | x=2022, y=2, n=30, z=Widget |

**Slash-encoded benchmark column** — same layout as Go bench names, stored in CSV:

```bash
vizb results.csv -g benchmark -p "[n/x/y]" -o out.html
```

| benchmark | Pattern | Result |
|---|---|---|
| `Sort/1024/QuickSort` | `[n/x/y]` | n=Sort, x=1024, y=QuickSort |

**Underscore-delimited ID:**

```bash
vizb runs.csv -g id -p "[n_y_x]" -o out.html
```

| id | Result |
|---|---|
| `lib_func_case` | n=lib, y=func, x=case |

**Dot-separated version on X/Y:**

```bash
vizb releases.csv -g version -p "[x.y]" -o out.html
```

| version | Result |
|---|---|
| `1.2.3` | x=1, y=2 (remaining segment unused) |

**Space-separated label in one column:**

```bash
vizb data.csv -g "label region" -p "[x n y],z" -o out.html
```

| label | region | Result |
|---|---|---|
| `alpha beta gamma` | `EU` | x=alpha, n=beta, y=gamma, z=EU |

**Prefix skip inside a bracket** (leading `/` drops the first segment of the cell value):

```bash
vizb data.csv -g name -p "[/n/y]" -o out.html
```

| name | Result |
|---|---|
| `BenchmarkTest/JSON/Marshal` | n=JSON, y=Marshal |

**Mix value-split with whole columns** — date parts + region on Z:

```bash
vizb sales.csv -g date,region -p "[n-y-x],z" -o sales.html
```

| date | region | Result |
|---|---|---|
| `2022-2-30` | `USA` | n=2022, y=2, x=30, z=USA |

**Comma value inside one column** (tuple-style cell):

```bash
vizb data.csv -g tuple -p "[y,x]" -o out.html
```

| tuple | Result |
|---|---|
| `USA,Widget` | y=USA, x=Widget |

**Mixed inner separators** (underscore then slash in the same cell):

```bash
vizb data.csv -g code -p "[n_y/x]" -o out.html
```

| code | Result |
|---|---|
| `Rivet_GPlusStatic/100k` | n=Rivet, y=GPlusStatic, x=100k |

<Aside type="note">
  Bracket splitting is **literal** — vizb does not parse dates or semver semantically. Values must match the separator layout in `-p`. For irregular strings without stable separators (e.g. `Hashing64MD5`), use [`--group-regex`](#regex-syntax) instead.
</Aside>

The columns you **don't** select decide the chart values:

- **Unselected numeric columns become series.** Here `sales` and `latency` each become a series (one `Stat` per column).
- **Non-numeric unselected columns are ignored.** A `notes` text column would simply be dropped.
- **No `--group` at all** → dimensions stay empty, no rows are merged, and each row becomes one point carrying every numeric column as a series. Useful for a quick chart of a wide table.

<Aside type="note">
  Each `--group` column must match a header exactly. A missing column is fatal: `group column 'X' not found; available: [...]`.
</Aside>

### Aggregation

When `--group` is set, multiple rows can share the same `(name, x, y, z)` key. Vizb **sums** their values into a single point so a 200k-row export collapses into a few thousand chart points. You'll see it in the CLI output:

```text
🧮 Aggregating 50000 rows...
✅ Aggregated into 1200 grouped data points
```

<Aside type="caution">
  Aggregation is **summing**, and applies to **CSV/JSON only**. Benchmark parsers are never aggregated — repeated runs share a key but the UI averages them instead of summing.
</Aside>

See [Tabular Data (CSV & JSON)](/guides/data) for the full parser rules.

## Grouping benchmarks

Benchmark names already encode structure. A name like `BenchmarkSort/1024/QuickSort` carries the function, the input size, and the variant. There are no columns to pick, so `--group-pattern` / `--group-regex` do all the extraction, and `--group` only supplies human-readable axis **labels** (see [Axis labels](#axis-labels-with---group---g)).

```bash
go test -bench . | vizb -p n/x/y -o output.html
```

| Benchmark Name | Name | XAxis | YAxis |
|---|---|---|---|
| `BenchmarkSort/1024/QuickSort` | Sort | 1024 | QuickSort |
| `BenchmarkSort/2048/MergeSort` | Sort | 2048 | MergeSort |

Every metric the benchmark reports (time, memory, allocations, …) becomes a series automatically — you only group the name.

## Pattern Syntax

The `--group-pattern` flag splits the label using the separators you declare between dimension tokens and assigns each segment to a dimension. It applies to both flows — the benchmark name or the CSV/JSON label built from `--group`.

### Separators

Any single character works between dimension tokens — vizb reads them from your `-p` string:

| Separator | Typical use |
|---|---|
| `/` | Go/Rust/JS benchmark names (`n/x/y`) |
| `_` | Underscore-delimited bench names (`n_y_x`) |
| `,` | Comma-separated `--group` lists (`y,x` with `-g region,product`) |
| ` ` (space) | Quoted `--group` specs (`"y x"` with `-g "region product"`) |

Consecutive separators (e.g. `x//y`) and a leading separator (e.g. `/n/y` to skip a prefix segment) are supported.

### Components

| Component | Shorthand | Meaning |
|---|---|---|
| `name` | `n` | Chart group name |
| `xAxis` | `x` | X-axis values |
| `yAxis` | `y` | Y-axis series |
| `zAxis` | `z` | Z-axis depth layer (3D, requires `x` + `y`) |

The default pattern is `x` — a single segment mapped to the X-axis.

### Examples

**Slash-separated `n/x/y`:**

```bash
vizb bench.txt --group-pattern "n/x/y" -o output.html
```

| Label | Name | XAxis | YAxis |
|---|---|---|---|
| `BenchmarkSort/1024/QuickSort` | Sort | 1024 | QuickSort |
| `BenchmarkSort/2048/MergeSort` | Sort | 2048 | MergeSort |
| `BenchmarkSort/4096/QuickSort` | Sort | 4096 | QuickSort |

**Underscore-separated `n_y_x`:**

```bash
vizb bench.txt --group-pattern "n_y_x" -o output.html
```

| Label | Name | YAxis | XAxis |
|---|---|---|---|
| `BenchmarkHash_SHA256_1KB` | Hash | SHA256 | 1KB |
| `BenchmarkHash_MD5_4KB` | Hash | MD5 | 4KB |

**2D with `n/y` (no X-axis grouping):**

```bash
vizb bench.txt --group-pattern "n/y" -o output.html
```

| Label | Name | YAxis |
|---|---|---|
| `BenchmarkJSON/Marshal` | JSON | Marshal |
| `BenchmarkJSON/Unmarshal` | JSON | Unmarshal |

**Prefix skip with `/n/y` (leading slash discards the first segment):**

```bash
vizb bench.txt --group-pattern "/n/y" -o output.html
```

| Label | Name | YAxis |
|---|---|---|
| `BenchmarkTest/JSON/Marshal` | JSON | Marshal |
| `BenchmarkTest/XML/Parse` | XML | Parse |

**4D with `n/x/y/z` (renders a 3D chart):**

```bash
vizb bench.txt --group-pattern "n/x/y/z" -o output.html
```

| Label | Name | XAxis | YAxis | ZAxis |
|---|---|---|---|---|
| `BenchmarkSort/1024/QuickSort/warm` | Sort | 1024 | QuickSort | warm |
| `BenchmarkSort/2048/MergeSort/cold` | Sort | 2048 | MergeSort | cold |

<Aside type="note">
  A pattern must contain `x` or `y` (`pattern must contain xAxis (x) or yAxis (y)`). A `z` segment requires both `x` and `y` — vizb rejects `z` otherwise with `zAxis (z) requires both xAxis (x) and yAxis (y)`. See [3D Charts](/charts/3d).
</Aside>

## Regex Syntax

The `--group-regex` flag uses Go-style named capture groups for full flexibility. It also runs on the label — a benchmark name, or the CSV/JSON label built from `--group`. Use it when the label does not follow a simple separator pattern. When set, regex takes precedence over `--group-pattern`.

### Named Capture Groups

| Group Name | Shorthand | Dimension |
|---|---|---|
| `(?<name>...)` | `(?<n>...)` | Chart group name |
| `(?<xAxis>...)` | `(?<x>...)` | X-axis values |
| `(?<yAxis>...)` | `(?<y>...)` | Y-axis series |
| `(?<zAxis>...)` | `(?<z>...)` | Z-axis depth layer (3D, requires `x` + `y`) |

### Examples

| Label | Regex | Extracted |
|---|---|---|
| `BenchmarkHashing64MD5` | `Hashing64(?<x>.*)` | X: MD5 |
| `BenchmarkJSONByMarshal` | `(?<x>.*)By(?<y>.*)` | X: JSON, Y: Marshal |
| `BenchmarkDecode/text=digits/level=speed` | `(?<n>.*)/text=(?<x>.*)/level=(?<y>.*)` | N: Decode, X: digits, Y: speed |

```bash
vizb bench.txt --group-regex "(?<n>.*)/text=(?<x>.*)/level=(?<y>.*)" -o output.html
```

<Aside type="note">
  The regex must capture `x` or `y` (`regex '...' does not contain x (xAxis) or y (yAxis)`), and a captured `z` still requires both `x` and `y`.
</Aside>

## Axis labels with `--group` / `-g`

`--group` carries human-readable labels for each dimension so charts render meaningful axis titles instead of raw identifiers. Labels map **positionally** to the `--group-pattern` parts, in order:

```bash
# Pattern n/y/x → --group labels: Sort=name, algorithm=y, size=x
go test -bench . | vizb -p n/y/x --group Sort,algorithm,size -o output.html
```

| Pattern slot | `--group` entry | Axis label |
|---|---|---|
| `n` (name) | `Sort` | chart group label |
| `y` | `algorithm` | Y-axis title |
| `x` | `size` | X-axis title |

The same flag plays a different role depending on the parser:

- **Benchmark parsers** (`go`, `rs:*`, `js:*`): `--group` is **labels only** — there are no columns. It names the dimensions that `--group-pattern` already extracted from the benchmark name.
- **CSV/JSON parsers**: `--group` entries are **column/field names** (or a structured spec with the same separators as `-p`). They select which columns build the label *and* serve as that dimension's label — one flag, both jobs. Comma-separated `-g` requires comma-separated `-p`.

<Aside type="note">
  Labels are included in the JSON output under `axisLabels` and displayed on the chart axes. The `name` label is preserved for the UI axis-swap feature, which lets you rotate dimensions while carrying their labels.
</Aside>

## Choosing a Method

<Tabs syncKey="method">
  <TabItem label="Pattern" icon="list">
    Use `--group-pattern` when the label follows a consistent separator format.

    ```bash
    # Benchmarks — slash-separated names
    vizb bench.txt --group-pattern "n/x/y" -o output.html

    # CSV/JSON — comma-separated columns
    vizb sales.csv -g region,product -p y,x -o sales.html
    ```

    Best for standard Go benchmarks (`BenchmarkFunc/size/variant`) and for CSV/JSON where columns map cleanly to dimensions with matching separators.
  </TabItem>
  <TabItem label="Regex" icon="search">
    Use `--group-regex` when the label has a non-standard format or you need fine-grained control over extraction.

    ```bash
    vizb bench.txt --group-regex "Hash(?<n>[A-Za-z]+)(?<x>\d+)" -o output.html
    ```

    Best for custom naming conventions or when you need to skip fixed prefixes and suffixes.
  </TabItem>
</Tabs>

<Aside type="tip">
  Either method splits the label the same way — so for CSV/JSON, remember you still need `--group` to pick the columns first. Benchmarks need no `--group` to chart, since the name is already the label.
</Aside>
