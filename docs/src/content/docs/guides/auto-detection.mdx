`--parser`/`-P` defaults to **`auto`**. When set to `auto` (or omitted), vizb inspects the input
**content** — not just the file extension — picks the best parser, and prints its choice:

```
✨ Auto-detected parser: csv
```

- Passing `-P <parser>` explicitly **skips** detection and prints nothing.
- Works for file arguments **and** piped stdin (detection runs on content).

## Detection order (strong → weak)

vizb tests signatures in priority order and returns the first match:

| Priority | Detected as | Signal |
|----------|-------------|--------|
| 1 | `go` | Go `go test -bench -json` event stream (JSONL with an `Action` field). |
| 2 | `json` | First non-whitespace byte is `[` (a JSON array). |
| 3 | `rs:divan` | Divan box-table rows (`├─`/`╰─`) or a `fastest │ slowest │ median` header. |
| 4 | `rs:criterion` | A `… time: [ … ns … ]` line. |
| 5 | `js:vitest` | A `·`-prefixed data row with ≥11 fields. |
| 6 | `js:tinybench` | A `console.table` with `Task name` / numbered `│ <n> │` rows. |
| 7 | `csv` | `.csv` extension, or a comma-separated table with ≥2 consistent columns. |
| 8 | `go` | Go benchmark text (`ns/op`, `goos:`/`goarch:`/`pkg:`/`cpu:`, `Benchmark…`). |
| 9 | `go` | Fallback (nothing else matched). |

## Safety

<Aside type="note">
  Detection is designed not to surprise you:

  - A `go test -bench -json` file named `.json` is **not** hijacked — it is detected as `go` and
    converted as before.
  - Vizb's **own** output JSON round-trips: it is consumed before any parser runs.
  - Detection is heuristic and bounded (it samples the first ~200 lines). A pathological input can
    always be forced with an explicit `-P`.
</Aside>

## Examples

```bash
vizb bench.txt                  # ✨ go      (benchmark text)
vizb data.csv                   # ✨ csv
vizb data.json                  # ✨ json    (array)  /  go (bench -json events)
cargo bench | vizb              # ✨ rs:divan or rs:criterion
go test -bench . | vizb         # ✨ go      (from piped content)
vizb data.csv -P csv            # no message (explicit)
```

[Tabular Data guide](/guides/data) for the CSV/JSON rules.
