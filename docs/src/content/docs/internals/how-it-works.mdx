Vizb converts benchmark output — or any tabular CSV/JSON data — into a structured JSON format, then embeds it in a self-contained HTML file with a Vue.js visualization app.

## Pipeline

```
go test -bench .  →  raw text  →  parser  →  Dataset struct  →  JSON / HTML
  Vitest output   ↘              ↓                       ↓
  Criterion output →         registry           (grouping applied)
  CSV / JSON      ↗      (format auto-detected)
```

## Data Structures

### Stat

A single metric value extracted from a benchmark result:

```json
{
  "type": "Execution Time (ns/op)",
  "value": 1523.4
}
```

### DataPoint

A single data point (benchmark entry or table row) with dimensions and stats:

```json
{
  "name": "Sort",
  "xAxis": "1024",
  "yAxis": "QuickSort",
  "stats": [
    { "type": "Execution Time (ns/op)", "value": 1523.4 },
    { "type": "Memory Usage (B/op)", "value": 256 },
    { "type": "Allocations (op)", "value": 4 }
  ]
}
```

### Dataset

The top-level output struct containing metadata and all data points:

```json
{
  "tag": "v1.1.0",
  "timestamp": "2025-01-15T10:30:00Z",
  "name": "MyBenchmarks",
  "description": "Sorting algorithm comparison",
  "history": [
    { "tag": "v0.9.0", "timestamp": "2024-12-01T08:00:00Z" },
    { "tag": "v1.0.0", "timestamp": "2025-01-15T10:30:00Z" }
  ],
  "cpu": { "name": "Apple M2", "cores": 8 },
  "os": "darwin",
  "arch": "arm64",
  "pkg": "github.com/example/sort",
  "settings": {
    "charts": ["bar", "line", "pie"],
    "sort": { "enabled": true, "order": "asc" },
    "showLabels": false,
    "scale": "linear"
  },
  "data": [
    { "name": "Sort", "xAxis": "1024", "yAxis": "QuickSort", "stats": [...] },
    { "name": "Sort", "xAxis": "1024", "yAxis": "MergeSort", "stats": [...] }
  ]
}
```

## Input Detection

Vizb auto-detects the input format:

<Tabs syncKey="format">
  <TabItem label="Raw Text">
  Standard `go test -bench` output. Parsed line by line using `golang.org/x/perf/benchfmt`.

  ```bash
  go test -bench . > bench.txt
  vizb bench.txt -o output.html
  ```
  </TabItem>
  <TabItem label="JSON Bench Events">
  `go test -bench -json` output. Vizb extracts the `output` field from each event and converts to text before parsing.

  ```bash
  go test -bench . -json | vizb -o output.html
  ```
  </TabItem>
  <TabItem label="Vizb JSON">
  Previously generated vizb JSON output. Loaded directly without re-parsing.

  ```bash
  vizb bench.txt -o data.json
  vizb data.json -o output.html
  ```
  </TabItem>
  <TabItem label="CSV / JSON Data">
  Generic tabular data. A CSV table or a JSON array of objects is parsed into data points — numeric columns/fields become series.

  ```bash
  vizb data.csv -o output.html
  vizb data.json -o output.html
  ```
  </TabItem>
</Tabs>

## Processing Steps

1. **Input** — file argument or stdin pipe written to temp file
2. **Preprocess** — JSON bench events converted to text (if needed)
3. **Detect Parser** — `DetectParser` inspects the content and picks a parser via the registry (Go/Vitest/TinyBench/Criterion/Divan/CSV/JSON), unless `--parser` forces one
4. **Parse** — the selected parser extracts names and stats from the input
5. **Group** — `--group-pattern` or `--group-regex` splits data labels into dimensions (n, x, y); `--group` feeds CSV/JSON columns in first
6. **Build** — assemble `Dataset` struct with metadata and settings
7. **Output** — JSON (`json.Marshal`) or HTML (Vue template + embedded JSON)
