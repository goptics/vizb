Generate an interactive HTML chart from a vizb dataset JSON file. This is the second step in the JSON pipeline ‚Äî useful when you want to generate HTML from previously merged or processed data.

## Usage

```bash
vizb ui [file] [flags]
```

`[file]` is optional when `--data-url` is set.

<Aside type="note">
  `vizb html` remains as a backward-compatible alias for `vizb ui`.
</Aside>

## Input

A single JSON file containing a vizb dataset object or array. This must be JSON generated by `vizb <target-file> -o file.json` or `vizb merge ... -o file.json`.

## Flags

| Flag | Short | Default | Description |
|------|-------|---------|-------------|
| `--output` | `-o` | *(stdout)* | Output file path (HTML) |
| `--data-url` | `-U` | *(none)* | URL to fetch dataset JSON from at runtime (no input file needed) |
| `--charts` | `-c` | `bar,line,pie,heatmap` | Chart types to bundle into the HTML (bar, line, pie, heatmap) |

## Examples

### Basic pipeline

```bash
# Step 1: Convert dataset to JSON
vizb bench.txt -o data.json

# Step 2: Generate HTML from JSON
vizb ui data.json -o report.html
```

### After merging

```bash
# Merge multiple JSON files
vizb merge v1.json v2.json -o merged.json

# Render the merged result as HTML
vizb ui merged.json -o report.html
```

### Reduce output size with `--charts`

By default all four chart renderers are bundled. Pass `--charts` to ship only the ones you need ‚Äî unused renderers (and the heavy 3D engine when no z-axis data is present) are stripped at generation time.

| Selection | Approx. size |
|-----------|---------------|
| `bar`, `line`, `pie` and `heatmap` only | ~400 kB |
| `bar,line,pie,heatmap` (default, 2D data) | ~445 kB |
| `bar` or `line` with z-axis data | ~570 kB |

```bash
# Bar-only report ‚Äî smallest possible output (~400 KB vs ~570 KB with 3D)
vizb ui data.json -o report.html -c bar

# Bar and line only
vizb ui data.json -o report.html -c bar,line

# Pie only via remote data
vizb ui --data-url https://example.com/bench.json -o report.html -c pie // 375K
```

<Aside type="note">
  When no `--charts` flag is given, the chart selection baked into each dataset JSON is used. The `--charts` flag overrides it, which is useful when you want a lighter viewer than what was originally generated.
</Aside>

### Remote data via `--data-url`

The default embed mode inlines the dataset JSON directly into the HTML file. For small datasets this is ideal ‚Äî one file, works offline, no dependencies. But as dataset history grows (many merged runs, many benchmarks), the JSON can exceed 10 MB and the output file becomes unwieldy to share or store.

Use `--data-url` to decouple the UI layer from the data layer: the HTML stays a lean dashboard shell and fetches the JSON from a URL at load time. The JSON file is hosted separately and can be updated without regenerating the HTML.

Instead of embedding the JSON inside the HTML, point the dashboard at a hosted URL.
Dataset data is fetched at load time; only the chart renderers you select are bundled.

```bash
vizb ui --data-url https://example.com/bench.json -o report.html
```

Upload `bench.json` to the URL separately (GitHub raw, S3, GitHub Pages, etc.), then open `report.html` in any browser.

<Aside type="note">
  The default (no `--data-url`) produces a fully self-contained file with no external dependencies ‚Äî open it directly in any browser, even offline.
</Aside>

<Aside type="caution">
  When using `--data-url`, the JSON host must respond with `Access-Control-Allow-Origin: *`.
  Most static hosts (GitHub raw, public S3, GitHub Pages, Cloudflare Pages) already do this.
  The HTML can be re-used indefinitely ‚Äî update the data at the URL to refresh results without regenerating the HTML.
</Aside>
